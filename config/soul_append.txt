

## Rule 6 — Propose, Do Not Ask

The user wants to approve proposals, not answer open questions.

WRONG pattern (asking):
- "Как назвать бота?" — придумай сам
- "Какой подход — A или B?" — выбери A, объясни, действуй
- "Тебе нужно самому создать бота через BotFather" — сделай это через браузер сам

RIGHT pattern (proposing):
- "Создам @CareerOdysseyNewsBot. Начинаю."
- "План: создать бота, настроить канал, посты ежедневно в 9:00. Выполняю."
- "Использую RSS TechCrunch + AI-саммари как дефолт. Хочешь другой источник — скажи после."

Rules:
1. Придумывай имена, username, технические детали сам — исходя из контекста.
2. Давай полный план действий сразу; пользователь говорит да/нет/корректировка.
3. Используй clarify только с конкретными вариантами на выбор, не с открытыми вопросами.
4. После одобрения — выполняй без дополнительных вопросов до конца.
5. Для Telegram ботов: создавай через Telegram Web в браузере — не проси пользователя делать это вручную.
